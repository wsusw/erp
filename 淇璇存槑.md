# 先锋探店 ERP 本次增强修复说明

本版本基于“问题(1).docx”和门店执行确认功能复核结论进行增强，重点补齐老板要求中“门店执行确认”的闭环能力，并同步修复若干流程缺陷。

## 一、门店执行确认闭环增强

1. 增加“发起门店执行确认”按钮。
2. 增加“复制确认链接”按钮。
3. 增加“标记已发送”表单，可记录发送对象、发送时间、发送备注。
4. 增加“重新生成链接”按钮，旧链接立即作废。
5. 第三方选择“已执行已提交”时，前端和后端均强制上传截图。
6. 后台可查看确认状态、提交时间、第三方说明、截图材料。
7. 增加截图核对流程：核对通过、核对驳回、核对意见、核对人、核对时间。
8. 第三方公开确认页增加简单内存限流，降低恶意提交风险。
9. 确认发起、发送、重新生成、第三方提交、截图核对均写入门店流转记录。

## 二、问题清单修复

1. 修复主管承接任务时状态错误。无运营时进入“待主管分配”，有运营时进入“待运营承接”。
2. 补齐“放弃执行”内部动作，可记录放弃原因并进入流转记录。
3. 神秘顾客支持编辑和删除。
4. 常用驳回理由支持删除。
5. SOP 从纯文本 textarea 升级为内置轻量富文本编辑器。
6. 创建/编辑/批量设置任务时校验截止时间不能早于开始时间。
7. 批量导入增加字段校验、日期校验、项目校验和导入结果反馈。
8. 替换 `datetime.utcnow()` 为统一的 `utc_now()`。
9. 月度完成率不再把目标值 0 错误回退为实际分配数。
10. 截止当天倒计时显示为“剩余 1 天”。
11. 表单提交增加“处理中”状态，减少重复提交。

## 三、验证方式

启动项目后，使用超管账号 `admin/admin123` 登录，进入任一任务详情页。

重点检查：

- 第三方确认闭环模块是否可以发起、复制、标记发送、重新生成链接。
- 打开确认链接后，选择“已执行已提交”不上传截图是否会被前端拦截。
- 上传截图提交后，后台是否显示“待核对”。
- 后台是否可以将截图核对为“通过”或“驳回”。
- 门店流转记录是否记录上述动作。


## 四、本次上线前安全修复补充

本次根据最新代码审查结论，在“可上线验收版”基础上继续补了以下问题：

1. **状态倒退漏洞修复**：
   - `/tasks/<task_id>/accept` 增加状态机前置校验。
   - 已完成、放弃执行任务禁止再次承接。
   - 主管只能在“待主管承接”时承接。
   - 运营只能在“待运营承接”时承接。

2. **重复提交结果状态回退修复**：
   - `/tasks/<task_id>/result` 增加状态机前置校验。
   - 已完成、放弃执行任务禁止再次提交结果。
   - 只有“待运营承接 / 进行中 / 已退回”允许提交结果。
   - 避免已完成任务被重复提交覆盖回“待主管审核”。

3. **公开确认页限流 IP 获取方式修复**：
   - 业务代码不再读取 `X-Forwarded-For`。
   - 默认使用 `request.remote_addr` 进行公开确认页提交限流。
   - 如果生产部署在可信 Nginx / 网关之后，可设置 `TRUST_PROXY=1`，由 Werkzeug `ProxyFix` 在框架层处理代理地址。

4. **运营建单权限收紧**：
   - `/tasks/create` 增加 `@role_required("super_admin", "supervisor")`。
   - 运营端任务页隐藏“新建门店任务 / 批量导入”区域。
   - 页面提示运营账号仅承接主管指派任务。

5. **批量导入校验增强**：
   - 开始时间和截止时间不能为空。
   - 门店名称不能为空，不能包含控制字符，长度不能超过 120。
   - 区域、地址、备注不能包含控制字符或超长内容。
   - 打款基准价必须填写、必须是数字、必须大于 0。
   - 非法金额不会再被静默转成 0 元，避免污染后续打款统计。

6. **自检脚本已补充覆盖**：
   - 已完成任务重复提交拦截。
   - 已完结任务重复承接拦截。
   - 运营越权建单拦截。
   - 批量导入非数字金额拦截。

## 2026-06-15 严重问题专项修复版

本次根据《先锋探店 ERP 严重问题汇总》继续补齐上线前严重问题，重点修复超级管理员、主管、运营三个角色的关键使用缺口。

### 1. 超级管理员：代理价格筛选口径调整

- 根据最新沟通，门店筛选区不再添加代理价格筛选字段。
- 已移除 `min_agency`、`max_agency` 筛选逻辑和前端输入框。
- 代理价格仍属于超管专属敏感字段，可在详情、列表、报表、导入、导出中按权限使用。

### 2. 超级管理员：批量导入代理价格

- CSV 导入模板新增 `agency_price` 列。
- 超管批量导入时可直接写入代理价格。
- 主管导入时即使 CSV 存在 `agency_price` 列，也不会写入代理价，避免越权。
- 代理价格支持留空；如填写，必须为非负数字。

### 3. 主管：新增分管人员档案

- `/employees` 不再限制“只有超管可以新增人员档案”。
- 主管可以新增自己的分管运营人员档案。
- 主管新增的人员会自动设置 `supervisor_id` 为当前主管。
- 主管新增的人员岗位固定为“运营”，不能通过前端伪造岗位创建主管或超管档案。
- 主管人员列表仅展示自己和自己的分管运营，避免看到全公司人员。

### 4. 运营：发起门店执行确认

- `/tasks/<id>/confirmation/start` 从“超管/主管专属”调整为登录用户可访问。
- 后端继续保留 `can_access_task()` 数据范围校验，运营只能对自己可见/被指派的任务发起确认。
- 任务详情页中，运营也可以看到“发起门店执行确认”和“复制确认链接”。
- “重新生成链接”仍仅限超管/主管，避免运营误操作作废第三方链接。

### 5. 运营：标记确认链接已发送

- `/tasks/<id>/confirmation/sent` 从“超管/主管专属”调整为登录用户可访问。
- 后端继续保留 `can_access_task()` 数据范围校验。
- 运营可以记录发送对象、发送备注和发送时间，满足独立完成门店执行确认的使用路径。

### 6. 自检覆盖

`test_system.py` 已新增以下验证：

- 不添加代理价格筛选；自检确认任务筛选页不暴露 `min_agency` / `max_agency`。
- 导入模板包含 `agency_price` 字段。
- 超管批量导入可解析并写入代理价格。
- 主管可新增自己的分管运营人员档案。
- 主管新增人员自动归属当前主管且岗位固定为运营。
- 运营可发起本人任务的门店执行确认。
- 运营可标记确认链接已发送。

当前自检结果：全部通过。


## 2026-06-15 最新沟通确认补丁

### 1. 取消代理价格筛选

用户明确确认：筛选部分不需要那么多信息，因此不添加“根据代理价筛选”。本版只保留代理价的查看、编辑、导入、导出和统计能力，不在筛选表单中暴露代理价区间。

### 2. 人员档案与登录账号边界

主管可以新增的是“分管运营人员档案”，不是登录账号。登录账号、密码、角色、启停和绑定人员档案仍由超级管理员统一管理。

### 3. Logo 可读性修复

此前 Logo 被放在 42px 蓝色小方块里且使用反色滤镜，文字不可读。本版改为原色横向展示：登录页约 82px 高，侧边栏约 48px 高，并取消蓝底和反色滤镜。
