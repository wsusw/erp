# 先锋探店 ERP 完整可执行版

这是根据《先锋探店 ERP 需求文档》和《UI 设计建议》整理并重做的一版完整 Flask 可执行项目。项目默认使用 SQLite，本地启动后会自动建表和生成演示数据，不需要手动初始化数据库。

## 1. 启动方式

### macOS / Linux

```bash
cd xf_erp_complete
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

也可以直接：

```bash
./run.sh
```

### Windows

```bat
cd xf_erp_complete
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

也可以双击或运行：

```bat
run.bat
```

启动后访问：

```text
http://127.0.0.1:5000
```

## 2. 默认账号

| 角色 | 账号 | 密码 | 权限说明 |
|---|---|---|---|
| 超级管理员 | admin | admin123 | 查看全部数据、账号管理、代理价、超管终审、全量导出 |
| 主管 | supervisor | supervisor123 | 查看分管范围、指派运营、审批 5-10 元加价、审批 15 元内路费、审核结果 |
| 运营 | operator | operator123 | 查看承接任务、确认承接、维护执行/收款信息、提交结果、提交价格和路费申请；不能主动新建任务 |

## 3. 已落地的核心业务逻辑

### 3.1 三级账号权限

- 超管：可以查看所有任务、人员、账号、项目、审批、门店确认、流转记录、代理价。
- 主管：只能查看自己分管范围内的任务和运营。
- 运营：只能查看自己承接的任务；历史版本中由运营补录的数据仍兼容显示，但新版严格按需求文档关闭运营主动建单入口。
- 代理价格只在超管账号下显示，主管和运营页面不渲染该字段。

### 3.2 任务指派链路

- 超管可新建任务并指派主管、运营。
- 主管可新建任务并指派自己分管的运营。
- 运营可录入任务，录入后进入待主管审核状态。
- 支持任务状态：待主管承接、待运营承接、进行中、待主管审核、已完成、已退回、异常上报、放弃执行。
- 支持主管和运营确认承接。

### 3.3 SOP 管理

- SOP 字段支持 HTML/富文本粘贴。
- 只有超管和主管可以编辑 SOP。
- 运营只能查看 SOP。
- 每次 SOP 编辑写入门店流转记录。

### 3.4 门店执行确认链接

- 每个任务自动生成唯一第三方确认链接。
- 第三方无需登录即可提交执行状态。
- 支持状态：已执行待提交、待执行、放弃执行、已执行已提交。
- 选择“已执行已提交”时必须上传 APP 提交成功截图。
- 提交后不可重复修改。
- 确认结果写入门店流转记录。

### 3.5 价格调整审批

- 打款价格 = 打款基准价 + 已通过加价。
- 减价直接通过。
- 加价小于 5 元自动通过。
- 加价 5 到 10 元由主管审批。
- 加价大于 10 元由主管通过后提交超管终审；超管也可以直接终审。
- 审批或驳回均写入门店流转记录。

### 3.6 路费补贴审批

- 运营、主管、超管均可在任务详情提交路费补贴申请。
- 15 元以内含 15 元由主管审批。
- 15 元以上由主管通过后提交超管终审。
- 审批可填写自定义理由，也可从账号设置中的常用驳回理由填充。
- 全部审批写入门店流转记录。

### 3.7 任务结果提交和审核

- 运营可以提交问题清单、执行说明、截图附件。
- 提交后任务进入“待主管审核”。
- 主管/超管可以通过、退回补充、异常上报。
- 通过后任务状态变为“已完成”。
- 退回或异常均保留审核意见，并写入流转记录。

### 3.8 神秘顾客管理

- 每个任务可添加多个神秘顾客。
- 记录姓名、手机号、身份备注、执行状态。
- 记录创建人和创建时间。

### 3.9 门店流转记录

- 新建门店、编辑基础信息、分配任务、编辑 SOP、编辑执行人/收款人、提交结果、审核结果、价格审批、路费审批、第三方确认等操作都会写入流转记录。
- 流转记录可在任务详情查看。
- 支持导出 CSV。

### 3.10 门店筛选、批量导入、批量设置、导出

- 支持按门店/地址/区域/工单号、状态、紧急度、起止日期、价格范围筛选。
- 支持下载 CSV 模板。
- 支持批量导入门店任务。
- 支持勾选任务后批量设置紧急度、起止时间、运营。
- 支持导出当前任务结果。

### 3.11 月度完成率和报表

- 人员档案里可设置月度应完成任务数。
- 报表页按月份计算运营完成率：月度实际完成任务数 ÷ 月度应完成任务数 × 100%。
- 超管可查看所有运营，主管可查看分管运营。
- 支持导出月度完成率 CSV。
- 报表页支持导出任务、流转、路费、第三方确认。

## 6. 生产部署提醒

这版是可执行验收版，适合本地演示、内部测试和继续二次开发。正式上线前建议：

1. 修改 `.env.example` 中的 `ERP_SECRET_KEY`。
2. 改用 MySQL 或 PostgreSQL。
3. 上传文件接入对象存储或公司文件服务器。
4. 增加 HTTPS、访问日志、备份策略。
6. 富文本编辑器可替换为 TinyMCE / Quill。
7. 对上传文件做更严格的病毒扫描和文件类型校验。
8. 对关键审批增加消息通知，例如企业微信、钉钉或短信提醒。

## 2026-06-15 严重问题专项修复说明

本版本在 safe release 基础上继续补齐严重问题清单中的具体问题：

- 按实际使用口径精简筛选项：不开发代理价格筛选，避免筛选区过重；代理价仍仅超管可见、可编辑、可导入、可导出。
- CSV 导入模板新增 `agency_price` 列，超管批量导入时可写入代理价。
- 主管可在人员管理中新增自己的分管运营人员档案。
- 主管新增人员时系统自动将直属主管设置为当前主管，岗位固定为运营。
- 运营可在自己有权限访问的任务详情页发起门店执行确认。
- 运营可标记确认链接已发送，并填写发送对象和备注。
- “重新生成确认链接”和“截图核对”仍保留给主管/超管，避免运营误作废链接或越权审核。

自检命令：

```bash
python test_system.py
```

自检覆盖本次新增权限、导入、门店确认和筛选精简修复项。


## 2026-06-15 最终沟通确认版

本版按最新沟通做了以下调整：

- 不添加“按代理价格筛选”功能，避免门店任务筛选区字段过多。
- 保留代理价格字段本身：仅超级管理员可查看、编辑、批量导入、导出和统计。
- 主管可以新增“分管运营人员档案”，不是新增登录账号；账号创建、密码和启停仍由超管在账号权限模块管理。
- Logo 展示尺寸已放大：侧边栏使用横向原色 Logo，登录页使用更大的原色 Logo，去掉小蓝色方块和反色滤镜。


## 本版新增：门店任务去项目化

- 新建门店任务不再需要选择项目。
- 批量导入模板已移除 `project_id` 列，只保留门店导入所需字段。
- 批量导入不再读取或校验 `project_id`，避免把序号误当项目 ID。
- 任务列表、任务详情、工作台和任务导出均去掉项目展示字段。
- 数据库模型中 `Task.project_id` 已改为可为空；如果你在旧版本数据库上直接覆盖代码，建议删除测试库 `xf_erp.db` 后重新启动，或执行数据库迁移。


## 本版补充

- 作废权限调整为：超管可作废所有未完结门店任务；主管可作废自己分管范围内的未完结门店任务；运营不可作废。
- Logo 展示改为黑底横向容器，增加缓存版本号和加载失败兜底文案，避免侧边栏/登录页空白。
